# Что ещё нужно реализовать в truemail

Дата повторной сверки: 2026-07-17.

В файле перечислены только подтверждённые пробелы текущей версии. Уже
реализованные пункты предыдущего аудита удалены.

## 1. Почта и провайдеры

### Поддержка провайдеров

- Для Outlook/Exchange Online пока нет рабочего OAuth-подключения.
- JMAP остаётся только вариантом модели данных: рабочего транспорта и
  диспетчеризации для него нет.

### Gmail push

- Почти реальное время Gmail сделано коротким polling последних ID. Для
  полноценного push всё ещё нужен отдельный механизм Gmail watch/Pub/Sub либо
  явно зафиксированное ограничение desktop-only архитектуры.

### Ещё отсутствующие почтовые функции

- Нет свайпов и drag-and-drop писем между папками.
- Нет автообновления приложения и защиты от запуска нескольких экземпляров
  (`tauri-plugin-updater` и `tauri-plugin-single-instance` не подключены).

## 2. Календарь и контакты

- Редактор события не позволяет задавать повторение, исключения, участников,
  организатора, напоминания и другие уже предусмотренные моделью экспертные
  поля.
- Перетаскивание события на другую дату/время отсутствует. Изменение даты
  возможно только через редактор.

## 3. Тесты и сборка

- Есть CI и 43 unit/integration теста, но основные сценарии outbox/retry, CRUD
  календаря/контактов и команды Tauri почти не покрыты.
- Нет сквозного теста: подключение тестового аккаунта -> sync -> чтение HTML и
  вложений -> отправка -> серверное перемещение/undo.
- На Linux/macOS `Makefile` использует shell-скрипт, но на Windows всё ещё
  требуется отдельно установленный `make`, которого нет в стандартной системе.
  Нужен нативный bootstrap-командлет/скрипт без обязательного GNU Make либо
  документированная установка инструмента.
