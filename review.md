# Что ещё нужно реализовать в truemail

Дата повторной сверки: 2026-07-17.

В файле перечислены только подтверждённые пробелы текущей версии. Уже
реализованные пункты предыдущего аудита удалены.

## 1. Почта и провайдеры

### Поддержка провайдеров

- Для Outlook/Exchange Online пока нет рабочего OAuth-подключения.
- JMAP остаётся только вариантом модели данных: рабочего транспорта и
  диспетчеризации для него нет.

### Gmail push

- Почти реальное время Gmail сделано коротким polling последних ID. Для
  полноценного push всё ещё нужен отдельный механизм Gmail watch/Pub/Sub либо
  явно зафиксированное ограничение desktop-only архитектуры.

### Ещё отсутствующие почтовые функции

- Нет свайпов и drag-and-drop писем между папками.
- Нет автообновления приложения и защиты от запуска нескольких экземпляров
  (`tauri-plugin-updater` и `tauri-plugin-single-instance` не подключены).

## 2. Календарь и контакты

- Редактор события не позволяет задавать повторение, исключения, участников,
  организатора, напоминания и другие уже предусмотренные моделью экспертные
  поля.
- Перетаскивание события на другую дату/время отсутствует. Изменение даты
  возможно только через редактор.

## 3. Интерфейс и UX

- Первый запуск по-прежнему показывает приветствие только после языка, выбора
  папки и создания ключей. Приветствие и объяснение процесса должны идти до
  создания хранилища.
- Для Яндекса OAuth всё ещё требует ручного копирования кода. Loopback callback
  реализован только для Gmail.
- Текст раздела тем обещает импорт тем, но UI и формат импорта отсутствуют.
- Горячие клавиши теперь честно показаны как фиксированные рабочие сочетания,
  но переназначать их нельзя; таблица `keybindings` не используется.
- Навигация всё ещё построена преимущественно на `div`. JavaScript добавляет
  `role`, `tabindex` и обработку Enter/Space во время выполнения, но нужен
  переход на семантические `button`/`a` и полноценная проверка экранным
  диктором.
- Локализация UI и core разделена между большим JS-словарём и Fluent-файлами.
  Нужен единый источник переводов и автоматическая проверка отсутствующих
  ключей/захардкоженных строк.
- `app.js` остаётся монолитом более чем на тысячу строк. Почту, композер,
  календарь, контакты, правила, настройки и i18n нужно разнести по модулям.

## 4. Схема и документация

- Не используются либо используются обходным путём таблицы `keybindings` и
  `image_trust`.
  Нужно либо подключить их как источник истины, либо удалить из схемы.
## 5. Тесты и сборка

- Есть CI и 39 unit/integration тестов, но основные сценарии outbox/retry, CRUD
  календаря/контактов и команды Tauri почти не покрыты.
- Нет сквозного теста: подключение тестового аккаунта -> sync -> чтение HTML и
  вложений -> отправка -> серверное перемещение/undo.
- На Linux/macOS `Makefile` использует shell-скрипт, но на Windows всё ещё
  требуется отдельно установленный `make`, которого нет в стандартной системе.
  Нужен нативный bootstrap-командлет/скрипт без обязательного GNU Make либо
  документированная установка инструмента.
## Приоритет

### P2

1. Семантическая доступность и единая локализация.
2. Модульная декомпозиция UI и расширение тестового покрытия.

### P3

3. Автообновление и single-instance.
4. Редактирование расширенных полей календаря и drag-and-drop событий.
5. Удаление или подключение оставшихся мёртвых таблиц и устаревших ссылок
    на документацию.
